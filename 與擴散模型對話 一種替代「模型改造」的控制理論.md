一種替代「模型改造」的控制理論

——基於語義繞行、潛在空間導航、退化理論與MAC的構想

---

## 一、問題：為什麼改了模型，問題還是沒解決？

當前AI繪圖社群的主流控制方法主要分為兩類：

**第一類：模型改造**
- LoRA微調、DreamBooth、ControlNet（部分屬於結構引導）
- 共同前提：生成失敗 = 模型能力不足

**第二類：輸入控制**
- Prompt工程、負面提示詞
- 共同前提：生成失敗 = 我沒寫對詞

但在實際使用開源擴散模型（SD、SDXL、Flux等）的過程中，可以觀察到一個現象：

**在不進行任何訓練的情況下，透過精確的語義結構與生成流程設計，模型仍然可以穩定生成高度一致的角色與場景。**

這引出一個不同的假設：

> **模型並非缺乏能力，而是使用者沒有有效引導模型的內部結構。**

因此本文提出一個替代方案：**生成控制本質上是一種與模型內部機制的對話**，而不是對模型的改造。

---

## 二、擴散模型的生成本質：從數學到直覺

為了理解這個對話框架，需要先明確擴散模型的生成過程。

**生成不是繪製，而是逐步從噪聲恢復結構。**

形式化描述為：
```
z_T → z_{T-1} → ... → z_0
```
其中 z_T 為純噪聲，z_0 為最終圖像的潛在表示。

這個過程可以理解為：**高維潛在空間向低維圖像結構的「坍塌」**。

在每一步，模型都在預測該去掉多少噪聲：

```
ϵ_θ(x_t, t, c)
```
- x_t：當前帶噪聲的圖像
- t：當前步驟（從T到0）
- c：條件變量（你的prompt）

模型透過以下目標函數學習：
```
E[ || ϵ - ϵ_θ(x_t, t, c) ||^2 ]
```
意思是：模型學會預測「圖像應該長什麼樣」，從而去掉那些「不該存在的噪聲」。

而實際更新圖像時，用的是這個公式：
```
μ_θ(x_t, t) = (1/√α_t) × ( x_t - ((1-α_t)/√(1-ᾱ_t)) × ϵ_θ(x_t, t) )
```
簡化理解：**每一步都在計算「下一步該往哪走」**。

在這個過程中，存在三個決定因素：
1. **語義條件（prompt c）**：決定「要去哪裡」
2. **注意力分配（attention）**：決定「哪裡是什麼」
3. **潛在特徵演化（latent trajectory）**：決定「怎麼走過去」

因此，**控制生成，本質上是對這三個層級的干預**。

## 三、一個更深的事實：退化理論與條件熵

這裡需要引入一個更深層的幾何結構。

模型的潛在空間有一個核心特性：**語義映射是多對一的**。意思是，大量不同的潛變量（不同seed、不同初始噪聲）會映射到相同或極相似的語義結果。

這個特性可以用**條件熵**來描述：

```
H(X|c) = -∫ p(c) ∫ p(x|c) log p(x|c) dx dc
```

什麼意思？

- 給定同一個prompt c，模型可能生成的圖像X仍然有不確定性
- 這個不確定性的大小，就是H(X|c)
- 當H(X|c)很大時，同一個prompt會產生差異巨大的結果
- 當H(X|c)很小時，結果非常穩定

**退化理論告訴我們**：這個不確定性不是bug，而是高維統計投影的必然產物。我們能做的不是消除它，而是**在給定c的情況下，最小化H(X|c)**——也就是，用更精確的c來降低不確定性。

這就是為什麼精確的特徵組合比模糊的角色名更有效：**它降低了條件熵，縮小了模型可能的輸出範圍**。

## 四、對話框架的三個層級

基於以上理解，我們提出一個三層控制模型：

### L1 語義對話：告訴模型「要去哪裡」

**對話對象**：cross-attention機制

當prompt被輸入模型時，語詞會被映射到attention map。這意味著：**語言本身已經是一種控制信號**。

在實踐中可以觀察到：語義拆解能顯著提高角色穩定性。例如將角色拆解為髮型、服裝、顏色、配件、比例，而不是直接使用角色名稱。

這一現象說明：**模型並不是單純記住角色，而是記住特徵分佈**。角色本質上是潛在空間中的一個概率區域，而非單一向量。這就是Semantic Bypass（語義繞行）有效的原因。

### L2 注意力對話：告訴模型「哪裡是什麼」

**對話對象**：attention map

注意力機制決定圖像中哪一部分對應哪個語義。近期提出的**MAC方法（Marginal Attention Constraints）**提供了重要證據。

MAC的核心思想是：透過監控attention map並施加約束來控制生成，具體包括三種損失：
- 區域約束：某個概念必須出現在某個區域
- 邊界約束：概念之間必須保持距離
- 存在性約束：某個概念必須出現（或不出現）

其特點是：**不需要重新訓練模型，只在生成過程中優化latent特徵**。

這證明了一個重要觀點：**模型內部已經包含空間結構，只需要正確引導即可**。

### L3 潛在空間對話：告訴模型「怎麼走過去」

**對話對象**：生成路徑（latent trajectory）

生成過程不是唯一的。從z_T到z_0存在多條可能的路徑，不同prompt結構會影響路徑選擇。

這可以從以下現象觀察到：**相同prompt、不同的順序，生成結果明顯不同**。prompt不只是條件，而是動態引導。

結合退化理論來理解：
- **纖維（Fiber）**：潛空間中映射到同一語義的區域。沿著纖維走，語義不變。
- **橫截（Transverse）**：跨越不同纖維的方向。沿著橫截走，語義變化。

不同的prompt結構，會讓模型選擇不同的路徑——有些沿著纖維走（結果穩定但相似），有些穿越橫截方向（結果有變化）。

因此可以提出：**Latent Space Navigation（潛在空間導航）**——透過語義順序與生成策略來控制坍塌路徑。

## 五、MAC與語義導航的關係

- **MAC解決的是局部問題**：物體位置、物體數量、區域一致性
- **語義導航解決的是全局問題**：角色身份、風格一致性、生成穩定性

可以用一個簡單的類比理解：
- MAC = 局部微調
- 語義導航 = 全局路徑規劃

兩者並不衝突，而是互補。一個完整的控制系統應包含：**全局生成策略 + 局部空間修正**。

## 六、LoRA的重新解釋

LoRA在實務上非常有效。從理論上看，它代表另一種策略：**修改模型分佈**。

這很高效，但需要有足夠的數據子樣本。這意味著當我們訓練LoRA時，本質上是在**重新塑造（或暴力破壞）潛在空間**。

這解釋了為什麼：
- 每個角色需要單獨的LoRA
- 不同模型之間難以遷移
- 可能影響其他生成能力

因此，LoRA可以被理解為：**「分佈修補」**，而不是「分佈導航」。

這也是為什麼對話方法具有更強的可遷移性——它不需要修改模型，只需要理解模型並與之對話。

---

## 七、開源與閉源模型的共同限制

在實際觀察中可以發現，開源乃至閉源模型存在以下問題：
- 角色體積感不足
- 光影過渡不自然
- 構圖容易扁平化

這可能源於潛在空間缺乏顯式幾何結構。即使在最先進的模型中，仍然存在相似限制。

**這說明問題不是模型大小，而是表示方式**。物理規則——空間深度、光線傳播、材質反射——這些是當前噪聲擴散模型暫且不具備的能力。不是模型「不夠強」，而是它的底層假設不同。

---

## 八、支持證據

以下現象支持對話理論的有效性：

1. **語義拆解可以替代角色LoRA**——用特徵組合就能穩定生成角色
2. **prompt順序會影響生成穩定性**——結構優先原則有效
3. **attention約束可以在不訓練模型的情況下改變結果**——MAC就是證明
4. **模型規模越大，對話方法效果越好**——大模型聽得更清楚

這說明：**模型能力早已存在，只是未被有效利用。**

---

## 九、結論

生成控制並不一定需要改造模型。

在很多情況下，更有效的方法是：**理解模型如何生成，並與其內部機制對話**。

三個核心方法：
- **語義繞行（Semantic Bypass）**：用特徵組合取代模糊標籤，降低條件熵
- **注意力約束（MAC）**：在生成過程中引導空間佈局
- **潛在空間導航（Latent Space Navigation）**：透過結構順序控制坍塌路徑

這三者共同構成了一種新的控制模式：

> **Prompt不再只是輸入文本，而是一種結構化控制語言。**

不是追求更大的模型，而是追求**更好的對話方式**。
