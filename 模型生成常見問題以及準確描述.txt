一、核心術語解釋 (附來源)

1. 泛化 (Generalization)

意思：模型學到的是這類東西的規律，而不是這張圖的細節 。
例如：模型看過很多蒼綠色雙馬尾的圖，它學到的是蒼綠色雙馬尾通常長什麼樣子、通常搭配什麼服裝、通常有什麼氣質 ，而不是記住某幾張特定的圖。
效果：能舉一反三，生成沒見過但合理的組合，有神韻、有靈氣、不會死板復刻。
來源：2025年Favero等人提出的“泛化時間” τ_gen 理論 (arXiv:2505.16959 )2025年Zhang等人從表徵學習角度證明泛化對應均衡表徵 (arXiv:2512.20963v1 )

2. 記憶 (Memorization)

意思：模型記住了訓練資料中的某個具體樣本或極小區域。
例如：模型看過大量某個角色的圖，它把這個角色的 精確特徵組合，鎖死在潛空間的一個小區域裡。
效果：結果穩定、細節準確、容易僵化，難以跳出。當你給的 prompt 和記憶區域重疊時，模型會強行對齊 。
來源：2025年Favero等人提出的 記憶時間 τ_mem 理論，證明 τ_mem 隨資料集大小線性增長 2025年Biroli等人從統計物理角度證明泛化先於記憶的現象 (NeurIPS 2025 Best Paper)

3. 平均 (Average)

意思：模型找不到明確的原型區域(archetype)時  取一個最安全，最不犯錯的折衷結果。
例如：你寫 a girl，模型不知道你要哪種女孩，就把所有女孩的特徵混在一起，給你一個平均臉、平均髮型、平均服裝 。
效果：穩定  低風險，平庸  模糊  沒細節。往往發生在  prompt  太模糊或特徵衝突時。
來源：擴散模型的條件分佈中心化特性，在多篇文獻中被稱為分佈平均化 (distribution averaging)或模式坍塌  (mode collapse)的前期表現。

4. 原型 (Archetype) / 原型吸引子 (Archetype Attractor)

意思： 潛空間中某個穩定的 高機率區域 ，對應某一類特徵的 典型代表 。
例如： 蒼綠色雙馬尾＋虛擬偶像 在潛空間中會形成一個原型區域，當你的 prompt 落在這個區域附近時，模型會自然被“吸引”過去。
效果：用特徵組合取代角色名，在導航到這個原型區域中，落在原型區域內但不鎖死時，生成結果會有“神韻”。 
來源：源於流形假說(Manifold Hypothesis)與吸引子動力學(Attractor Dynamics)2025年Biroli等人的研究將此概念形式化為泛化窗口內的穩定結構。 


二、為什麼有時候你需要負面 prompt ？

1. 用來過濾技術性缺陷 

負面 prompt 最有效的場景，是過濾那些與語義無關、純粹是生成缺陷的東西。
例如：no text, no watermark, no extra fingers, no deformed hands, no blur, no low quality.
這些技術性缺陷在模型訓練資料中常常與好圖混在一起，用負面 prompt 可以簡單地把它們壓低機率 。

2. 用來 拉開距離 避免混淆

當你的正面 prompt 中出現多個容易混淆的概念時，可以用負面 prompt 把它們拉開。
例如：你想要 A 角色的服裝 + B 角色的髮型，但模型總是偏向 B 角色。這時可以在負面 prompt 裡寫 [B角色]，讓模型不要滑過去 。

3. 用在不確定原型區域邊界時，如果你不確定某個原型區域的邊界在哪，先用負面 prompt 把你不想要的東西擋在外面，是一個安全做法。


三、什麼時候不需要 負面 prompt？

1. 正面 prompt 已經精確鎖定原型 

當你用精確的特徵組合 (例如 teal twin-tails, black pleated skirt, virtual idol )鎖定了你想要的原型區域。
模型自然會落在這個區域內  此時負面 prompt 不僅多餘，還可能因為先理解才能排除的機制，反而引入不必要的干擾。

2. 你想要 神韻 或 泛化空間 
如果你留了一點模糊空間，讓模型在原型區域內自由發揮，這時加負面 prompt 反而會把這個自由空間壓縮掉，讓結果變得僵硬。

3. 負面 prompt 與正面內容衝突
例如正面寫 warm sunlight，負面寫 no shadow——模型會陷入矛盾，結果往往是兩邊都做不好，或取一個奇怪的折衷。


四、要不要用負面 prompt

過濾技術缺陷 (文字、浮水印、多手指 )  要用 避免概念混淆 (A 與 B 容易混 )  可以用
正面 prompt 已精確鎖定“原型”不需要。
你想要神韻或泛化空間 不需要。 
負面與正面內容衝突 不要用。


常見問題以及現象、解析原因、附方案(適宜進階用戶)

(給進階用戶的速覽版)

1 模型很強但結果很“平”，取平均。不是鎖子集，用特徵組合取代標籤，給精確座標
2 泛化、記憶、平均怎麼選 ？三者共存，取平均是失控，想要靈氣留泛化，想要穩定鎖記憶
3 構圖前期有效後期無效；早期定結構，後期補細節，構圖寫在前面，或用 ControlNet 釘死
4 負面 prompt 可能反向翻車模型先理解才能排除，正面鎖準原型，負面只用技術過濾
5 突然出現“神韻”落在泛化子集，不是平均，適度留白，讓模型在原型區域內發揮

問題一：為什麼現在的模型明明很強，卻常常生成很“平”的結果  ？

現象：
同樣一段 prompt，早期模型(如 SD 1.5)生成的結果常常糊、散、不精準；現在的大模型(SDXL Flux)明明能力更強啊？
但有時候生成的圖反而很平——不是構圖平，而是“特徵被平均了”。比如你想要蒼綠色雙馬尾，它卻給你一個蒼綠色雙馬尾的平均值，細節全沒了。

原因：
這不是模型變差，而是模型變強的方式變了。
早期模型：數據少、容量小，只能學到大致方向，所以結果泛、散、不夠準。
現在的大模型：數據量巨大，它已經能記住大量具體的原型區域(也就是潛空間子集  )。
但當你給的 prompt 不夠精確時，模型會默認取這個區域的平均結果——把各種相似特徵混在一起，給你一個最安全、最不犯錯的版本。
所以不是模型畫不好，而是它沒收到足夠精確的定位指令。

方案：
不要只寫角色名或風格名，用特徵組合去鎖定你想要的子集。
把 prompt 當成結構說明書，而不是購物清單。
如果想讓模型聽你的，就要給它足夠窄、足夠具體的“潛空間座標”。


問題二：泛化、平均、記憶，到底是互斥還是互補 ？

現象：
有時候你寫的 prompt 並不精確，但模型反而生成了一張“很有靈氣”的圖；有時候你寫得特別細，結果卻死板、僵硬、像套模板。
這些無理由的情況讓人困惑並沮喪。到底該讓模型泛化多一點，還是記憶多一點？

原因：
這三者不是非黑即白的關係。
泛化：模型學到這類東西的規律，能舉一反三，有靈氣。
記憶：模型鎖定某個具體子集，結果穩定、細節準，但容易僵化。
平均：模型找不到明確方向，取一個  最安全  的折衷，結果平庸。
在好的生成裡泛化和記憶是可以共存的——泛化負責感覺，記憶負責特徵鎖定。
而強行對齊則是記憶過強時，模型硬把你說的東西往它記最熟的子集上靠(比如你說  A 角色的服裝 + B 角色，它直接給你 B 角色)。

方案：
想要“有靈氣但穩定”：用精確特徵鎖定角色原型，但留一點空間給泛化。
想要“穩定復刻” ：用 LoRA 或極精確的特徵描述，強行鎖死。
想要“跨界組合” ：要主動拉開距離，比如先鎖定主體特徵，再單獨描述服裝，避免被單一強記憶綁架。


問題三：為什麼 prompt 對構圖的控制力，前後期不一樣？

現象：
你常常會發現一開始就寫清楚構圖(平視、居中、側光)，出來的圖結構就很穩。
但如果生成到一半才改構圖，基本沒用。
甚至有時候你覺得我明明寫了構圖，它怎麼還是亂來？  

原因：
擴散模型的生成過程是從高維雜訊向低維圖像塌縮。
早期(大步數)：語義主導 決定：這是什麼東西、整體怎麼擺。 ——這時候構圖、主體、光影方向一旦定下來，後面很難改。
後期(小步數)：細節主導 注意力集中在紋理、邊緣、局部光影上；所以構圖控制必須在“塌縮”中期完成，後期改它根本不聽。

方案：
把構圖、視角、光線方向寫在 prompt 最前面。
不要指望先生成再微調構圖，要在生成前就想好。
想精細控制構圖用 ControlNet(Canny/Depth/OpenPose)在早期就把結構釘死。


問題四：負面 prompt  (Negative Prompt)真的有用嗎 為什麼“no NSFW”反而會翻車？

現象：
很多人習慣寫負面 prompt 來防踩雷，例如 “no NSFW, no bad hands, no blur”。
但有時候你寫了 “no NSFW”，結果反而生成的東西可能會更危險；或者負面 prompt 寫了一大堆，該出的問題照樣出。

原因：
負面 prompt 的機制不是“禁止”，而是降低機率  。
更麻煩的問題是：當模型看到 “no NSFW”時，它需要先理解什麼是 NSFW，才能在後續步驟裡降低它  。
在早期雜訊階段，這個理解本身就可能把模型帶偏，反而加強了不該有的方向。
如果正面 prompt 已經精確鎖定某個乾淨原型，負面 prompt 不僅多餘，還可能添亂。

方案：
如果正面 prompt 已經足夠精確(特徵鎖死)，負面 prompt 可以不用，或少用。
負面 prompt 更適合用來過濾技術性缺陷，例如“text, watermark, extra fingers”。
不要用負面 prompt 去對抗正面內容，那容易造成內部衝突。


問題五：為什麼有時候模型生成的圖，會突然有“神韻”

現象：
你寫了一個不算特別細的 prompt，結果出來一張圖——說不上哪裡像，但就是對味，有情緒、有感覺，像某個角色但不是復刻。
這種圖往往比你寫得很細的圖更打動人。

原因：
這時模型沒有掉進平均區，也沒有死板地複製某個訓練樣本。
它落在了這個角色的泛化子集裡——也就是原型吸引域。
在這個區域，模型保留了原型的核心特徵，但允許局部自由組合，所以生成結果既有身份感，又有新鮮感。
這就是所謂的   “神韻”  ，本質是泛化，不是平均。

方案：
如果你想要穩定復刻，用精確特徵鎖死。
如果你想要有“神韻”，留一點模糊空間，讓模型在原型區域內自己發揮。
控制好鎖定和留白的比例——鎖太死，僵硬；太鬆，跑偏。
